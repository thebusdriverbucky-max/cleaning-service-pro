# План реализации (Todo List) — Дополнительный функционал клининга

Данный список задач предназначен для реализации в режиме **Code** после согласования архитектуры.

## Шаг 1: Обновление схемы базы данных Prisma
- [ ] Добавить модель `ServiceAddon` в `prisma/schema.prisma`.
- [ ] Добавить поля цен за комнаты (`pricePerBedroom`, `pricePerBathroom`, `pricePerKitchen`) в модель `ServiceType`.
- [ ] Добавить поля комнат (`bedroomsCount`, `bathroomsCount`, `kitchensCount`) и JSON-поле `addons` в модель `CleaningOrder`.
- [ ] Запустить `npx prisma db push` для синхронизации локальной базы данных.
- [ ] Сгенерировать клиент Prisma (`npx prisma generate`).
- [ ] Написать скрипт сидирования или обновить существующий в `prisma/seed.ts` для добавления тестовых доп. услуг (помыть холодильник, духовку, окна, глажка) и привязки цен комнат к типам услуг.

## Шаг 2: Создание констант и утилит ценообразования
- [ ] Добавить константы скидок по умолчанию в `lib/constants.ts` (Weekly: 20%, Biweekly: 15%, Monthly: 10%).
- [ ] Создать общую функцию расчета стоимости в `lib/prices.ts` (или `lib/utils.ts`), чтобы использовать одну и ту же формулу на клиенте (BookingForm) и на сервере (API).

## Шаг 3: Обновление API создания заказов
- [ ] Изменить `app/api/booking/create/route.ts`:
  - Получать новые поля из JSON: `bedroomsCount`, `bathroomsCount`, `kitchensCount`, `frequency`, `addons` (массив ID).
  - На сервере загружать цены выбранных допов из БД (для исключения фальсификации цен).
  - Применять формулу расчета: базовая + комнаты + допы -> скидка частоты -> промокод.
  - Сохранять выбранные допы в виде массива JSON в поле `addons` заказа `CleaningOrder`.
  - Записывать детальное количество комнат.
  - Передавать правильную итоговую сумму в Stripe сессию (включая описание выбранных комнат и допов).
  - Обновить шаблон подтверждающего письма (Email), добавив туда информацию о комнатах, периодичности и выбранных допах.

## Шаг 4: Разработка интерфейса формы бронирования (`components/booking/BookingForm.tsx`)
- [ ] **Шаг 1 формы (Service Selection)**:
  - Добавить раскрывающийся аккордеон "What is included?" с текстом из `service.description`.
  - Добавить стилизованные переключатели для выбора периодичности (`ServiceFrequency`) с информацией о скидках.
- [ ] **Шаг 2 формы (Booking Details)**:
  - Считывать цены на комнаты (`pricePerBedroom`, `pricePerBathroom`, `pricePerKitchen`) из выбранной услуги.
  - Если цены заданы, заменить обычное поле ввода комнат на интерактивные счетчики `[-] N [+]` для спален, ванных и кухонь.
  - Выводить секцию "Customize Your Clean (Add-ons / Extras)" с карточками-чекбоксами. Подгружать допы, подходящие для данной услуги или глобальные.
- [ ] **Шаг 3 формы (Payment & Сводка)**:
  - Обновить калькулятор на клиенте, используя общую утилиту расчета.
  - Красиво и подробно детализировать чек (Subtotal, Rooms, Addons, Frequency Discount, Promo, Total).

  - Считывать денормализованный JSON `addons` и отображать список выбранных дополнительных услуг с их ценами.
